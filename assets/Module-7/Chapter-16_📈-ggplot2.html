4 What is ggplot2?

Open the help file for ggplot() to see its formal arguments and description.

?ggplot   # skim the Details section

📜 The SYNTAX (basic/general):

ggplot(data = NULL, mapping = aes(), ..., environment = parent.frame())

The three core steps of ggplot2 are: (1) supply a data frame, (2) map aesthetics with aes(), and (3) add geometric objects (geoms) with geom_*() functions.

6 Mapping Aesthetics

Aesthetic mapping is the process of connecting variables in your data to visual properties of the plot, such as position, color, size, and shape. In ggplot2, this is done using the aes() function.

Review the syntax for ggplot2 again.

📜 The SYNTAX (basic): ⁠ggplot(data = df, mapping = aes(x, y, other aesthetics))

The aes() argument supplies mapping information. Mapping refers to connecting your data variables to visual properties (aesthetics) of the plot. It’s how you tell ggplot which column in your data should control which aspect of the visualization.

?aes

🎯 The GOAL: Map divorce rate to the x-axis and margarine consumption to the y-axis using aes().

# ⚡Code Example: Mapping aesthetics
divorce_margarine %>%
  ggplot(aes(x = divorce_rate_maine, y = margarine_consumption_per_capita))

🗣 Comment: The plot is still blank because we haven’t added any geometric objects (geoms) yet. The axes are now labeled by the mapped variables, even if there are no points yet. aes() connects data columns to visual positions. The aes() step comes before choosing a geom because we are first defining the mapping (what goes on the x- and y-axes, colors, etc.) and then deciding how to draw those mapped variables (points, lines, bars, etc.).

🪞 Reflection: The aesthetics describe the underlying structure of the plot, and the geoms are just different ways of rendering that structure. By setting aes() inside ggplot() we create a shared mapping that all later geoms can reuse, instead of repeating ourselves for each layer.

7 Adding Geometric Objects (Geoms)

Geometric objects (geoms) are the actual shapes drawn on the plot: points, lines, bars, boxplots, etc. Each geom answers the question: Given these aesthetics, how should the data be drawn?

7.1 Scatterplot Example

Scatterplots are used to visualize the relationship between two numeric variables. Each point represents an observation in the data, with its position determined by the values of the two variables.

geom_point() is a geometric object (or “geom”) in ggplot2 that creates a scatter plot by placing points at the specified x and y coordinates.

?geom_point

📜 The SYNTAX:

geom_point(mapping = NULL, data = NULL, ..., na.rm = FALSE, show.legend = NA, inherit.aes = TRUE)

🧐 Notice that all of the arguments are have default settings because geom_point() can inherit the data and aesthetic mappings from the initial ggplot() call.

👉 When using ggplot2, geoms are added to the plot using the + operator.

  1. 🎯 The GOAL: Add points to the plot using geom_point().
# ⚡Code Example: Scatterplot
divorce_margarine %>%
  ggplot(aes(divorce_rate_maine, margarine_consumption_per_capita)) +
  geom_point()

🗣 Comment: There are points showing a linear, positive association between divorce rate and margarine consumption per capita.

  1. 🎯 The GOAL: Map additional aesthetics inside geom_point(): color by year, size by divorce rate, and transparency (alpha) by margarine consumption.
divorce_margarine %>%
  ggplot(aes(x = divorce_rate_maine,   # Global aes mapping: x-axis
             y = margarine_consumption_per_capita)) + # Global aes mapping: y-axis
  geom_point(aes(color = year))        # Local aes mapping: color points by year

🤔 “Wait”, you might by thinking, “aes() is already defined in ggplot(), why are we using it again in geom_point()?” The aes() in ggplot() are global mappings for the entire plot and apply to all layers. When we use aes() inside a specific geom like geom_point(), we’re defining mappings that are specific to that layer. This allows for more granular control over how different layers of the plot behave.

geom_point() understands these aesthetics:

  • x, y: Position (required)

  • alpha: Transparency

  • color: Point color

  • fill: Fill color (for shapes with fill)

  • shape: Point shape (0-25)

  • size: Point size

  • stroke: Border thickness

🚀 Explore and Play: Try adding any of the above aesthetics to geom_point(). How does it change the plot?

7.2 Layering geoms

Different geoms tell different stories. Let’s explore more geoms!

We’ve seen that geom_point() highlights individual data points. geom_smooth() adds a smoothed line to show trends in the data.

?geom_smooth

📜 The SYNTAX:

geom_smooth(mapping = NULL, data = NULL, method = NULL, se = TRUE, ...)

Look at the argument method: This controls the smoothing method used. Common options include:

  • “lm”: Linear model (straight line)

  • “loess”: Local regression (curved line)

🎯 The GOAL: Add geom_smooth() to the scatterplot.

Since the relationship looks linear, we’ll use method = "lm" to fit a straight line. We’ll also set se = FALSE to hide the confidence interval around the line for clarity.

# ⚡Code Example: Scatterplot with trend line
divorce_margarine %>%
  ggplot(aes(x = divorce_rate_maine, 
             y = margarine_consumption_per_capita)) +
  geom_point(aes(color = year)) +          # Scatter points
  geom_smooth(method = "lm", se = FALSE)   # Add linear trend line

🗣 Comment: The trend line shows a positive linear relationship between divorce rate and margarine consumption.

7.3 Histogram Example

Histograms are used to visualize the distribution of a single numeric variable. They group values into bins and show the count (or density) of observations in each bin.

?geom_histogram

📜 The SYNTAX:

geom_histogram(mapping = NULL, data = NULL, ..., binwidth = NULL, bins = NULL, na.rm = FALSE, show.legend = NA, inherit.aes = TRUE)

Let’s build our own data set to explore histograms.

  1. 🎯 The GOAL: Simulate a Tallness data frame with heights for 1,000 people in the USA and Canada each. In the column Country, make two groups labeled “USA” and “CAN”. USA heights are centered around 67 inches with a standard deviation of 2.5. CAN heights are centered around 66 inches with a standard deviation of 1.5. Tallness follows a normal distribution.
# ⚡Code 
set.seed(2025)
Tallness <- data.frame(
  Country = c(rep("USA", 1000), rep("CAN", 1000)),
  Height  = c(rnorm(1000, 67, 2.5), rnorm(1000, 66, 1.5))
)
# ✅ Verify Tallness data frame
nrow(Tallness)
Tallness %>% 
  group_by(Country)%>%
  summarize(n = n(),
            mean_height = mean(Height),
            sd_height = sd(Height),
            .groups = "drop")

🗣 Comment: We have 2,000 rows, 1,000 for each of the two country labels, and heights roughly around expected means and standard deviation.

  1. 🎯 The GOAL: Build a histogram of heights using geom_histogram().
# ⚡Code Example: Histogram
Tallness %>%
  ggplot(aes(Height)) +    # only x for histograms
  geom_histogram()

🗣 Comment: Histograms visualize the distribution of a single continuous variable. geom_histogram() uses the global x aesthetic (Height) and so there is no need to repeat the aes() inside of the geom_histogram() argument. Additionally, the y is computed, not mapped. geom_histogram() uses stat_bin() “under the hood” to compute the counts (or density). This is why we do not need to map a column to y. The y-values are produced by the statistic.

geom_histogram() understands these aesthetics:

  • x: Position (required) and usually entered in ggplot(aes())

  • fill: Fill color

  • color: Border color

  • binwidth: Width of each bin (default is 1)

  • position: Position adjustment (e.g., “identity”, “dodge”, “fill”)

  • alpha: Transparency

  • size: Border thickness

  • weight: Weight for each observation (optional)

  • linetype: Line type for borders (optional)

  1. 🎯 The GOAL: Customize the histogram by setting fill, color, and binwidth.
# ⚡Code Example: Customized Histogram
Tallness %>%
  ggplot(aes(Height)) +                # only x for histograms
  geom_histogram(fill = "steelblue", 
                 color = "white",      # white border gives gap between bars
                 binwidth = 0.5)       # Set bin width to 0.5

🗣 Comment: The histogram shows a unimodal distribution roughly centered in the high 60s.

🚀 Explore and Play: Change binwidth = 0.25 inside geom_histogram(). Are the overall center and spread the same, even though the “smoothness” changes? Reflect on how binwidth trades off smooth vs. noisy appearance and what you’d choose for a report.

7.4 Density Plot

Now we’ll look at a smooth version of the height distribution using a density curve.

Density plots are smoothed versions of histograms that estimate the probability density function of a continuous variable. They are useful for visualizing the distribution shape without the binning artifacts of histograms.

?geom_density

📜 The SYNTAX:

geom_density(mapping = NULL, data = NULL, ..., na.rm = FALSE, show.legend = NA, inherit.aes = TRUE)

🎯 The GOAL: Make a smooth density plot of heights in the Tallness data and emphasize that the vertical axis is density, not counts. Add a density layer that shows a smooth curve for the distribution of heights. Fill the area under the curve, make it partially transparent so the shape is not too heavy.

# ⚡Code Example: Density Plot
Tallness %>%
  ggplot(aes(x = Height)) +
  geom_density(fill     = "purple",
               alpha    = 0.4,
               color    = "purple",
               linewidth = 1)

geom_density() uses kernel density estimation to create a smooth curve that represents the distribution of the data.

  • x is the variable whose distribution you want to smooth and is usually entered in ggplot(aes()).

  • geom_density() computes the vertical axis as density by default; the y-values are not mapped from a column.

  • fill sets the interior color of the density shape.

  • alpha controls transparency (0 = fully transparent, 1 = solid).

  • color sets the outline color.

  • linewidth controls the thickness of the outline.

7.5 Overlaid Histograms

Let’s compare the two height distributions by overlaying histograms and using fill = Country to distinguish groups.

🎯 The GOAL: Compare height distributions between USA and CAN by overlaying two histograms using fill = Country. Let’s store the graph as tall_compared_fig. Then, we’ll print the figure to take a look.

# ⚡Code Example: Overlaid Histograms
tall_compared_fig <- Tallness %>%
  ggplot(aes(Height, fill = Country)) +
  geom_histogram(position = "identity", 
                 alpha = 0.6)           # Set transparency to see overlap
# ✅ Print the figure
print(tall_compared_fig)

🗣 Comment: fill = Country in the global aes() tells ggplot() to make a separate histogram for each country with different fill colors, and automatic a legend. By default, geom_histogram() stacks groups. The position = "identity" instead puts the bars in the same x-scale so they overlap. The alpha = 06 makes bars transparent (0 = invisible, 1 = solid), so we can see where the two distributions overlap.

7.6 Boxplot

Boxplots summarize distributions using five statistics: minimum, first quartile (Q1), median, third quartile (Q3), and maximum. They are useful for comparing distributions across groups.

?geom_boxplot

📜 The SYNTAX:

geom_boxplot(mapping = NULL, data = NULL, ..., outlier.color = NULL, outlier.shape = 19, outlier.size = 1.5, na.rm = FALSE, show.legend = NA, inherit.aes = TRUE)

We can use boxplots to compare height distributions between USA and CAN.

🎯 The GOAL: Build side-by-side boxplots of height by country using geom_boxplot(). Map Country to the x-axis and Height to the y-axis. Use fill = Country to color the boxes by country and set show.legend = FALSE to hide the legend. We’ll store the boxplot as Tall_Box.

# ⚡Code Example: Boxplot
Tall_Box <- Tallness %>%
  ggplot(aes(Country, Height, fill = Country)) +
  geom_boxplot(show.legend = FALSE)
# ✅Print the boxplot
Tall_Box

🗣 Comment: There are two boxes, one for each country, with USA maybe a bit higher median and wider spread. The countries appear on the x-axis and that each box shows median, IQR, and dots for each of the outliers.

8 Labels and Themes

We can enhance our plots by adding axis labels and applying themes for better aesthetics.

xlab() and ylab() functions set the labels for the x and y axes, respectively. Simply type the desired label text as a string inside the function.

Additionally, ggtitle() adds a title to the plot.

Type ?theme_bw to explore available themes in ggtheme {ggplot2}

🎯 The GOAL: Add axis labels and a light theme to the boxplot of height by country.

# ⚡Code Example: Boxplot with labels and theme
Tall_Box <- Tallness %>%
  ggplot(aes(Country, Height, fill = Country)) +
  geom_boxplot(show.legend = FALSE) +
  xlab("North American Country") +
  ylab("Height (inches)") +
  ggplot2::theme_light()
# ✅Print the boxplot
Tall_Box

🗣 Comment: xlab() and ylab() set axis labels. ggplot2::theme_light() applies light horizontal grid lines. (The ggplot2:: prefix explicitly specifies that you’re using the theme_light() function from the ggplot2 package. This is useful when multiple packages have functions with the same name.)

10 Chapter Terms

Aesthetic mapping: A connection between a variable in the data and a visual property in the plot, such as position, color, or fill. In ggplot2, mappings are usually set inside aes().

Boxplot: A graph that summarizes a distribution using the median, quartiles, spread, and possible outliers.

Density Plot: A graph for continuous numerical data that shows a smoothed estimate of the distribution of values. In ggplot2, a density plot is typically created with geom_density() and can be understood as a smooth alternative to a histogram.

ggplot2: An R package for data visualization based on the grammar of graphics. It builds plots by combining data, mappings, and geometric layers.

geom: A geometric layer that controls how data are drawn on the plot. Examples include geom_point(), geom_histogram(), and geom_boxplot().

Histogram: A graph that shows the distribution of one numeric variable by grouping values into bins.

Labels: Text added to a ggplot2 graphic to identify and explain what the plot shows, such as axis labels, legend titles, plot titles, subtitles, captions, and tags. In ggplot2, labels are commonly set with labs(), xlab(), ylab(), and ggtitle()

Scatterplot: A graph that shows the relationship between two numeric variables by placing one point for each observation.

Theme: A set of display settings that controls the non-data parts of a ggplot2 graphic, such as fonts, background, gridlines, axis text, legend appearance, and title styling. Complete themes like theme_bw() or theme_light() change the overall look of a plot, while theme() lets you customize individual parts.
