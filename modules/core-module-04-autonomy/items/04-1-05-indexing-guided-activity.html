Purpose

Bracket indexing is one of the main ways analysts point R at exactly the rows, columns, and values they need.

In this guided activity, you will load a small dataset from CSV and use [ ], [[ ]], and $ to subset, extract, and modify data in precise ways.

That work is worth doing now because later analysis depends on selecting the right pieces of a dataset, understanding what type of object your code returns, and catching problems like missing-value propagation before they distort your results. These skills will support later filtering, recoding, column creation, and reproducible notebook work in R.

Chapter

Data

Dataset: field_sightings.csv

This guided activity uses a simulated bird-banding dataset with two columns, Status and Band_Color, to practice positional indexing, names indexing, logical masks, extraction, and subassignment.

Values_Transport_dplace.csv appears in the chapter setup because it is used later in Practice Space, but it is not the main dataset for this Guided Activity.

Guiding Questions

  • How does the meaning of [ ] change when you use positions, column names, or a logical mask?
  • When you subset one column from a data frame, what is the difference between field_sightings["Band_Color"], field_sightings[, "Band_Color"], and field_sightings[, "Band_Color", drop = FALSE]?
  • What is the practical difference between [ ], [[ ]], and $ when you need the column itself for a function like unique() or table()?
  • How can NA values inside a logical mask change a subset in ways you may not expect?
  • As you follow the insertion examples, how do you tell whether your code is overwriting existing values or creating a new column?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.
