Purpose

Review the supplemental resources on this page if you would like an alternative explanation, example, or walkthrough for concepts in this chapter or need extra help with a specific skill.

These materials are optional and do not replace the chapter.

Resources

  • Import Data Cheatsheet
    • This cheatsheet is most useful as a quick guide to the import functions themselves, especially read_csv(), read_delim(), read_tsv(), and read_excel(), plus arguments like col_names, skip, n_max, na, and col_types when a file does not import cleanly. It is the page to keep open when you need the exact syntax for choosing a delimiter, reading only part of a file, or fixing guessed column types.
  • Data Science: A First Introduction — Chapter 2 Reading in Data Locally and from the Web
    • This reading is most useful for understanding relative versus absolute file paths and why relative paths usually make imports work more reliably across different computers and project folders. It also helps clarify how the working directory affects imports, including practical path pieces like . for the current folder and .. for the previous folder.
  • Data Analysis and Visualization with R for Genomics: R Basics Continued — Factors and Data Frames
    • This reading is most useful for checking what you imported after the file opens, especially with dim(), summary(), and str(). It helps you recognize common data frame issues by showing how to read row and column counts, how to inspect each column’s type, and how to notice when character data, integers, and other columns were imported differently than expected.
  • Importing Tabular Data into R
    • Use this section when you need a more detailed look at read.csv() and the arguments that commonly need adjustment during import, such as header, sep, dec, and nrow. It is especially helpful for understanding that read.csv() is a shortcut built on read.table(), which matters when a file has an unusual delimiter or decimal mark.

Video

Importing Excel Files

This video is most useful if you need a practical walkthrough of importing Excel files into RStudio, especially when the data are not sitting neatly on the first sheet or in the top-left corner.

It is especially helpful for seeing how read_excel() uses arguments like sheet, range, col_types, and na to pull in the right cells, define missing values, and clean up import problems such as duplicate column names or messy working-directory setup.

If the video above does not appear, use the direct link: Importing data from excel into R studio (14:56 min.).
