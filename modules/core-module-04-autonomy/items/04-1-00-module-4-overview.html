🧭 Autonomy

In this Core module, autonomy means being able to manage data work without waiting for every step to be handed to you. You will practice two skills that make independent analysis possible: importing data into R and selecting the exact rows, columns, or values you need. You will work with CSV and Excel files, check file locations with getwd() and list.files(), load packages, import datasets with read_csv() and read_excel(), and use bracket indexing to subset, extract, and modify data.

The first Core topic focuses on getting data into R correctly. You will learn why the working directory matters, how package and built-in datasets differ from external files, and how to check that R can see the files you plan to import. The second Core topic focuses on indexing. You will use [ ], [[ ]], and $ to select parts of a data frame, work with logical masks, handle possible NA behavior, and correct data-entry problems through subassignment.

Across the module, you will keep using an R Notebook workflow: write code, check output, memo what you noticed, and save files that can be reopened and reviewed. The pattern is practical: first learn the workflow in a Guided Activity, then use Practice Space to show that you can apply the same ideas on your own. The main goal is not just to import or index once; it is to build a reliable workflow for finding files, loading data, selecting data carefully, and documenting what changed.

Learning Objectives

By the end of this module, the successful student will be able to:

  • Describe tabular data using rows as observations and columns as variables.
  • Use getwd() and list.files() to check where R is looking for files.
  • Load required packages with library() and distinguish package datasets from external files.
  • Import CSV and Excel files into R using read_csv() and read_excel().
  • Apply bracket indexing with df[rows, cols] to subset rows and columns by position, name, or logical condition.
  • Distinguish the output of [ ], [[ ]], and $ when extracting data from a data frame.
  • Explain how NA values can affect logical masks and why which() can help produce cleaner row indices.
  • Use subassignment to modify selected values in a copied data frame while preserving the original object.
  • Create reproducible notebook work that includes code, checks, memo notes, and saved output files.

Why This Topic is Important

Real data work starts before analysis. If R cannot find your file, if the wrong file is imported, or if you do not notice that a column imported differently than expected, every later step becomes unreliable. This module helps prevent one of the most common beginner problems: treating file-management errors as mysterious R errors. Checking the working directory, confirming visible files, and reading import feedback are basic habits that save time and prevent avoidable mistakes.

Indexing is just as important because most analysis does not use a whole dataset all at once. You often need selected rows, selected columns, one extracted vector, or a corrected copy of a data frame. This module teaches you to point R at the exact part of the data you mean, then verify that the result is the object type and size you expected.

These skills connect directly to later modules. Once you can import data and select parts of it carefully, you are better prepared for filtering, recoding, summarizing, plotting, and writing reports. The autonomy in this module is practical: you are learning how to manage files, inspect imported data, make careful selections, and document changes without guessing.

Strategies for Success

  • Keep your notebook and data files in the same course folder unless the chapter tells you otherwise.
  • Run getwd() and list.files() before importing a file; do not guess where R is looking.
  • Watch import messages for rows, columns, delimiters, and guessed column types.
  • Use clear object names such as dplace_df, car_prices_df, and values_transport so you can tell imported objects apart.
  • After every subset, check the result with functions such as head(), names(), nrow(), ncol(), unique(), or table().
  • Pay attention to whether your code returns a data frame or a vector; this matters for what functions will work next.
  • Use which() when you need row positions from a logical condition and want to avoid confusing NA behavior.
  • Make a copy of a data frame before overwriting values so the original imported data stays available for checking.
  • Memo what you checked and why, especially when file paths, import steps, or indexing output did not behave as expected.
