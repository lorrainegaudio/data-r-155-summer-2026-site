Purpose

This Practice Space checks whether you can apply the main bracket-indexing skills from Chapter 8: Bracket Indexing after completing the associated Guided Activity.

In this chapter, you load a CSV, subset rows and columns with base R indexing, extract a column as a vector, explain missing value propagation, and overwrite incorrect values using subassignment.

This practice matters because analysts rarely use an entire dataset at once. They need to point R at the exact rows and columns they need, check their results, and keep a reproducible record of what they changed.

What This Practice Space Checks

  • Use readr::read_csv() to load Values_Transport_dplace.csv into R as a data frame.
  • Apply bracket indexing with df[rows, cols] to subset data by position, by column name, and by a logical condition.
  • Explain missing value propagation in a logical mask and why which() helps avoid confusing subsetting results when NA values may appear.
  • Extract one column as the thing itself using [[ ]] so it can be checked with functions such as table() or unique().
  • Use subassignment with df[rows, "col"] <- value to overwrite incorrect values in a copied data frame.

Task 5

⚠️ Explain Missing Value Propagation

🎯 Goal: Explain what missing value propagation means when you create a logical mask like this:

mask <- values_transport[, "year"] >= 1900

If there were NA values in the year column, what will R return when a comparison involves NA, such as NA >= 1900? Why can NA values in a logical mask cause confusing subsetting results with [mask, ]? How does which() help avoid this issue?

When you create a logical mask like mask <- values_transport[, "year"] >= 1900, what happens if values_transport[, "year"] contains NA values?

  • When a comparison involves NA, R returns NA, so the mask can contain NA values that may produce unexpected rows unless you use which() to keep only TRUE indices.
  • When a comparison involves NA, R returns FALSE, so the mask never contains missing values and subsetting is always safe.
  • When a comparison involves NA, R returns 0, so the mask becomes numeric and subsetting fails unless you convert it back to TRUE/FALSE.
  • When a comparison involves NA, R returns NA, so using which() keeps TRUE and NA positions to prevent losing data.
