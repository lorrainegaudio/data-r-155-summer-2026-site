Purpose

Importing data is how analysts move from toy examples to real project workflow.

In this guided activity, you will learn how tabular data is organized, how the working directory affects file imports, and how to load built-in, package, CSV, and Excel datasets into R using a simple, reliable notebook workflow.

That work is worth doing now because many import errors are really file-management errors, and later analysis depends on getting data into R correctly before you can explore, subset, summarize, or visualize it. These skills will help you build a notebook that can Knit from top to bottom and reload your data from a clean start.

Guided Activity

  • Time estimate: 1 hr 30 min
  • Do: Monday
  • Open: Chapter_7_📤ImportingData.html
  • Use: RStudio, an R Notebook, the readr, readxl, and dslabs packages, and the chapter data files downloaded from this site.
  • Have ready: Your course folder, Values_Transport_dplace.csv and car_prices.xlsx saved in the same folder as your notebook, and a new R Notebook saved as chapter7_practice.Rmd.
  • Produce: A working R Notebook with follow-along code, memo notes, file checks, package loading, and successful dataset imports that can Knit from top to bottom.

Chapter

  • Chapter_7_📤ImportingData.html
  • Focus on the guided activity sections on tabular data, working directories, downloading and organizing files, checking file visibility with getwd() and list.files(), loading packages, and importing datasets with data(), read_csv(), and read_excel().
  • Pay close attention to the difference between loading built-in or package datasets and importing external files from your course folder.

Data

Datasets:

  • Values_Transport_dplace.csv
  • car_prices.xlsx

This guided activity uses two external files so you can practice importing both CSV and Excel data into R.

The chapter also uses built-in and package datasets such as mtcars and movielens to distinguish between loading data that already comes with R or a package and importing files from your own folder.

Guiding Questions

  • How does the chapter distinguish between tabular data, rows, columns, observations, and variables?
  • Why does the working directory matter when you Knit a notebook and try to import a file by name?
  • What checks in the chapter help you confirm that R can actually see the files you downloaded before you try to import them?
  • What is the difference between loading a dataset with data() and importing a file with read_csv() or read_excel()?
  • When you import a file, what does the feedback about rows, columns, delimiter, and column types help you notice about whether the import worked correctly?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.

Before You Leave This Page

  • Confirm that your CSV and Excel imports run without file-path errors.
  • Confirm that your notebook Knits from top to bottom from a clean start.
