Purpose

This assignment collects your Chapter 8 Bracket Indexing Practice Space work as a rendered HTML report and a saved workspace file.

For this chapter, those uploaded files matter because they show how you used bracket indexing to load data, subset rows and columns by position and name, handle logical masks carefully, extract a column as a vector, and correct data-entry errors in a reproducible workflow that can be reviewed and rerun.

File Upload

  • Required files to submit: chapter8_practice.html, chapter8_practice.RData
  • File type: .html, .RData
  • These files should show:
    • your completed Chapter 8 Practice Space work on bracket indexing using Values_Transport_dplace.csv
    • the actual R code used for Task 1 through Task 8
    • your positional, names-based, and logical-mask subsetting work, plus extraction with [[ ]] and overwrite work with df[rows, "col"] <- value
    • your chapter-specific objects, including values_transport, first_10_basic, id_name_year, values_1900, transport_year_clean, and mask
    • your checks after code, such as names(), head(), nrow(), unique(), and table() where used in the tasks
    • your learning memos and short written responses for each task
  • Based on: Chapter 8: Bracket Indexing
  • If applicable, include: headings for each task, YAML with the title, your name, and the date, and brief disclosure of outside help in a memo if you used it.

How You Will Be Marked

✅ Complete if all are true

  1. Code + tasks
    • The uploaded chapter8_practice.html shows the completed Task 1 through Task 8 for Chapter 8.
    • The work is clearly for Chapter 8 bracket indexing, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The report shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific subsetting work: loading the CSV, creating first_10_basic, creating id_name_year, creating values_1900, extracting the year column as a vector, and correcting the year-entry errors in transport_year_clean.
    • The code shows bracket indexing with row and column positions, character column names, logical comparisons, which(), [[ ]], and subassignment.
  2. Memos
    • The file includes learning memos or short written responses for each required task.
    • The memo responses answer the chapter’s specific prompt questions, including the size of first_10_basic, what id_name_year contains, how many entries have year values of 1900 or later, what missing-value propagation means in a logical mask, what you notice from the year extraction output, and why a copy was made before overwriting year values.
    • The memos explain what the code showed or confirmed, not just what code was typed.
    • If outside help was used, that help is briefly disclosed in a memo.
  3. Reproducibility / chapter-specific requirements
    • The submission includes the correct output files: chapter8_practice.html and chapter8_practice.RData.
    • The HTML report rendered successfully and opens as a complete report.
    • The .RData file includes the required saved objects: values_transport, first_10_basic, id_name_year, values_1900, transport_year_clean, and mask.
    • The code includes the required data-loading step with readr::read_csv() and the needed package call for readr.
    • The workflow includes checks after code so the subsets, extracted values, and corrected year values can be verified.
    • The files document the chapter workflow clearly enough that the bracket-indexing work can be reviewed and rerun later.

  • No attachment.
  • Wrong artifact.

  • The .RData file does not include the required saved objects.

  • The report does not include the required written responses or memo notes.
