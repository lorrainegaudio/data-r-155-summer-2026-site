Purpose

Review the supplemental resources on this page if you would like an alternative explanation, example, or walkthrough for concepts in this chapter or need extra help with a specific skill.

These materials are optional and do not replace the chapter.

Resources

AI Prompting

  • Marking UP the Prompt by Neuralbuddies
    • This reading is most useful for seeing how Markdown structure can change the quality of an AI prompt, especially by using headers, lists, bold text, and code blocks to separate instructions, examples, and output format. It is a good fit for this chapter if you need help understanding when formatting improves a prompt and when plain language matters more.
  • Boosting AI Performance by Anupam Mukherjee
    • Use this reading to focus on the comparison between Markdown and heavier formats like JSON or XML when the goal is readable prompt content rather than machine-only data exchange. It is especially useful for understanding why headings, lists, and simple structure can make source material easier for an LLM to interpret.
  • How To Write Effective AI Prompts by Daniel Miessler
    • This reading is most useful for practical prompt-writing moves: giving the model a role, breaking work into small steps, being explicit about the output format, and giving examples when needed. It also reinforces why many people use Markdown to keep prompts easy to read and revise.
  • OpenAI: Message Formatting with Markdown and XML
    • Choose this reading when you need a more formal guide to structuring prompts with sections such as Identity, Instructions, Examples, and Context. It is especially useful for learning how Markdown headers and lists plus XML tags can mark boundaries between instructions, examples, and reference material.

R Markdown

This video is most useful if you need a practical introduction to the R Markdown workflow from start to finish: creating a document, understanding the YAML header, adding text formatting and headings, inserting R code chunks, and then knitting to HTML.

It is especially helpful for seeing how chunk options like echo, eval, and include change what appears in the final report, and for watching simple additions such as lists, links, images, and a basic table.

If the video above does not appear, use the direct link: What is R Markdown? (15:14 min.).

R Markdown: Short

This video is most useful if you want to see the basic R Markdown in RStudio workflow with a simple analysis: create an .Rmd file, choose an output format, add plain text plus R code chunks, and then knit the document to HTML.

It is especially helpful for watching how code, output, and plots stay together in one report while doing practical steps like summary(), pairs(), and fitting a simple lm() model.

R Markdown: Long

This video is most useful if you need a beginner-friendly walkthrough of the R Markdown workflow for learning R, not just for reporting results.

It is especially helpful for seeing how to organize notes with headings, insert R code chunks, knit to HTML, add a table of contents, and keep track of errors or sessionInfo() so the document becomes a reusable study guide as well as a working record of code.

AI Prompting

It is especially helpful for seeing practical prompt moves such as asking for headings, ordered lists, bullet lists, bold text, and even a Markdown table so the response is more structured instead of one long block of text.
