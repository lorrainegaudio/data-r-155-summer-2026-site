💪 Tenacity

This module introduces the basic habits you need to work in R without getting lost: installing and opening RStudio Desktop, navigating the RStudio panes, running commands, assigning values to objects with <-, and saving your work. You will move from temporary Console commands toward a more durable workflow where your files, objects, and workspace can be checked and reused.

The 🌱 Core work focuses on setup, navigation, and scripts. You will practice using the Console, Environment, Source pane, working directory, .RData workspace files, and R scripts. You will also begin creating vectors and a simple data frame while checking your objects with functions such as length(), mode(), seq(), c(), and data.frame().

The 🌳 Advanced work extends the module by asking you to reflect on why you are learning R, connect course habits to your own goals, and participate in a small-group discussion. This module sets the pattern for the course: complete Core first, check your work carefully, revise when needed, and choose Advanced activities when you want to build toward a B or A bundle.

Learning Objectives

By the end of this module, the successful student will be able to:

  • Navigate the main RStudio panes, including the Console, Source pane, Environment, Files, and Help areas.
  • Use basic R syntax for arithmetic, object assignment with <-, object naming, and Environment cleanup.
  • Apply built-in functions such as sum(), mean(), sd(), sqrt(), getwd(), ls(), rm(), save.image(), and load().
  • Create and save an R workspace file as a reproducible snapshot of required objects.
  • Write and run an R script from the Source pane instead of relying only on temporary Console commands.
  • Build vectors with c(), seq(), and :, then check them with length() and mode().
  • Create a simple data frame with data.frame() from vectors created earlier in the script.
  • Use comments and short memos to document errors, troubleshooting steps, and what code output means.
  • Describe personal goals for learning R and connect them to course habits such as persistence, reproducibility, and careful documentation. (🌳 Advanced)

Why This Topic Is Important

Most beginner R problems are not caused by advanced statistics. They come from simple workflow issues: typing in the wrong place, losing track of objects, saving files in the wrong folder, using inconsistent object names, or relying on Console history instead of a saved script. This module helps you build the habits that prevent those problems before the course becomes more complex.

Good R work depends on being able to rerun what you did. Scripts, comments, working directories, and saved workspace files make your work easier to check, revise, and explain. These skills also make it easier to spot mistakes because you can look back at the exact code that created an object or output.

The Advanced work matters because learning R requires persistence. You will make errors. The goal is not to avoid every mistake; the goal is to notice what happened, test a fix, document what you changed, and keep going. That is the practical meaning of tenacity in this module.

Strategies for Success

  • Install and open R and RStudio Desktop before you try to complete later tasks.
  • Use a dedicated course folder from the beginning. Do not scatter files across Downloads, Desktop, and random folders.
  • Pay close attention to object names. In R, items_sold, Items_sold, and items sold are not the same thing.
  • Run code in small chunks first, then test the full script from top to bottom.
  • Use comments as working notes, especially when something breaks and you have to troubleshoot.
  • Check your Environment after creating objects. Do not assume an object exists just because you typed code once.
  • Save the required file type exactly as requested, especially .R and .RData files.
  • Compare your output to the task goal before submitting. A file that opens is not automatically a file that meets the specifications.
