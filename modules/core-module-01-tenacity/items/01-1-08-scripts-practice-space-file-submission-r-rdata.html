01.1.08 📑 Scripts Practice Space File Submission R and RData

🌱 Core | 30 min | Module 1: 💪 Tenacity

Purpose

This assignment collects your Chapter 2 Scripts Practice Space work as an R script file and a workspace file.

For this chapter, those uploaded files matter because they show that you can write and save a script in the Source pane, run it from top to bottom without errors, build and check several kinds of vectors, create a simple data frame, and document your troubleshooting process with comments and tenacity notes.

File Upload

  • Required files to submit: chapter2_practice.R, chapter2_practice.RData
  • File types: .R, .RData
  • These files should show:
    • your completed Chapter 2 Practice Space work for Task 0 through Task 7
    • the actual R code used in your script
    • your vector-building work with exponents, list_numbers, nerdy_palette, color_names, and origin
    • your data-frame work with nerdy_df
    • your checks after code, including length(), mode(), and other chapter checks
    • your task comments and any tenacity memo notes if you had to troubleshoot errors
  • Based on: Chapter 2: Scripts
  • If applicable, include: comments that explain the output for the required checks and a brief tenacity note describing how you resolved any errors you encountered.

How You Will Be Marked

✅ Complete if all are true

  1. Code + tasks
    • The uploaded chapter2_practice.R contains the completed Task 0 through Task 7 for Chapter 2.
    • The work is clearly for Chapter 2 Scripts, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The script shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific objects exponents, list_numbers, nerdy_palette, color_names, origin, and nerdy_df.
    • The script includes the required chapter work with seq(), :, c(), length(), mode(), and data.frame().
  2. Memos
    • The file includes the required task comments where the chapter asks you to explain the output.
    • The comments explain what the checks showed, including vector length, mode, stored color codes, appended values, and the counts of names or origins.
    • If you encountered errors, the script includes a brief tenacity note explaining how you troubleshot the problem.
    • If outside help was used, that help is briefly disclosed in a memo or comment.
  3. Reproducibility / chapter-specific requirements
    • The submission includes the correct files: chapter2_practice.R and chapter2_practice.RData.
    • The script runs from top to bottom when you source the entire file after sweeping the Environment.
    • The latest edits are saved before closing RStudio.
    • The .RData file contains the main Practice Space objects created in the script, including exponents, list_numbers, nerdy_palette, color_names, origin, and nerdy_df.
    • The files document the Chapter 2 workflow clearly enough that the script can be reviewed and rerun later.

🟡 Not Ready if any are true

  • No attachment.
  • Wrong artifact.
  • The file cannot be opened.
  • Only one required file is submitted.
  • The script is blank, incomplete, or unrelated to Chapter 2.
  • The required objects or checks are missing.
  • The script does not run from top to bottom because of errors.

If your submission is marked 🟡 Not Ready, revise to address the missing elements and resubmit for reevaluation before the assignment locks.

❌ Missing

No submission.

Before You Submit

  • Confirm your script is named chapter2_practice.R.
  • Confirm your workspace is named chapter2_practice.RData.
  • Confirm both files are uploaded.
  • Confirm the script includes Tasks 0–7.
  • Confirm the script runs from top to bottom after clearing the Environment.
  • Confirm your comments or memos explain the required checks and any troubleshooting.
