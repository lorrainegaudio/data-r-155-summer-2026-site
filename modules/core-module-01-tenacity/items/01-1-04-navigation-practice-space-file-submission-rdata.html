01.1.04 Navigation Practice Space File Submission RData

Core | 30 min | Module 1: Tenacity

Purpose

This assignment collects your Chapter 1 Basic R Commands Practice Space work as a saved workspace file.

For this chapter, the uploaded file matters because it shows that you can start with a clean Environment, store basic objects correctly, compute with those objects, and save your R session work in a form that can be reopened and checked.

File Upload

  • Required file to submit: chapter1_practice.RData
  • File type: .RData
  • This file should show:
    • your completed Chapter 1 Practice Space work from a clean Environment
    • the saved workspace objects created in the Practice Space, including items_sold, unit_price, and total_revenue
    • the correct stored values for those objects
    • evidence that you completed the object-storage and calculation steps before saving your workspace
  • Based on: Chapter 1: Navigation
  • If applicable, include: only the saved workspace file for this chapter. This file upload does not require an HTML report, notebook, or script submission.

How You Will Be Marked

Complete if all are true

  1. Code + tasks
    • The uploaded chapter1_practice.RData is clearly for Chapter 1 Basic R Commands Practice Space, not a different chapter or assignment.
    • The saved workspace contains the Chapter 1 practice objects items_sold, unit_price, and total_revenue.
    • items_sold is stored as 62.
    • unit_price is stored as 100.
    • total_revenue is stored as the product of those two objects.
    • The file is readable, opens correctly, and is not blank or replaced with unrelated content.
  2. Memos
    • No separate memo document is required for this specific file upload.
    • If you used outside help while completing the chapter, follow any course directions for disclosure elsewhere, but this upload is graded mainly on the saved workspace objects.
  3. Reproducibility / chapter-specific requirements
    • The submission is the correct file format: chapter1_practice.RData.
    • The workspace was saved after starting from a clean Environment, as directed in the Practice Space.
    • The file contains the needed Chapter 1 objects rather than an unrelated or cluttered workspace.
    • The saved workspace is specific enough that the instructor can reopen it and verify the Chapter 1 object work quickly.

Not Ready if any are true

  • No attachment.
  • Wrong artifact.
  • The file cannot be opened.
  • The required objects are missing or incorrect.
  • The uploaded file is unrelated to Chapter 1 Practice Space.

If your submission is marked Not Ready, revise to address the missing elements and resubmit for reevaluation before the assignment locks.

Missing

No submission.

Before You Submit

  • Confirm your file is named chapter1_practice.RData.
  • Confirm your Environment includes items_sold, unit_price, and total_revenue before saving.
  • Confirm you are uploading the .RData file, not a screenshot, script, notebook, or HTML file.
