Purpose

Scripts matter because they turn one-off commands into a saved record of what you ran, in what order, and why.

In this guided activity, you will practice writing and saving an R script, running code from the Source pane into the Console, and using comments as memos to document your thinking and troubleshooting.

That work is worth doing now because later analysis depends on having code you can re-run, check, and explain instead of relying on memory or a temporary Console history. These skills will also prepare you for later work with vectors, object checks, and data frames in a more reproducible R workflow.

Guided Activity

  • Time estimate: 1 hour
  • Do: Wednesday
  • Open: Chapter_2_📑Scripts.html
  • Use: RStudio, an R script in the Source pane, and example objects created directly in R.
  • Have ready: Your course folder open in RStudio and a new R script saved in that folder as chapter2_practice.R so you can write code, add comments, and test it from top to bottom.
  • Produce: A saved chapter2_practice.R file with follow-along code, comments that memo your thinking and troubleshooting, vector creation and checks, and a simple data frame built from vectors.

Chapter

  • Chapter_2_📑Scripts.html
  • Focus on the main guided activity sections on the Source pane versus the Console, creating and saving an R script, running code in order, using comments as memos, building vectors with c() and seq(), checking objects with mode() and length(), and creating a data frame with data.frame().
  • Pay close attention to the order-of-execution example and the reminder to test the whole script from top to bottom.

Data

No external dataset required.

This guided activity uses numeric, integer, character, and logical vectors that you create in the script, then combines same-length vectors into a simple data frame called spectrum_shift.

The chapter is about script workflow and basic object creation, not importing a separate dataset.

Guiding Questions

  • How does the chapter show the difference between typing commands in the Console and saving them in an R script?
  • What do RStudio’s Run tools make possible when you want to test one line, a block of lines, or the full script?
  • How do comments help you document not just what your code does, but also what you tried when something failed?
  • What checks with mode() and length() help you confirm that a vector is the kind of object you think it is?
  • How does the chapter move from creating basic vectors with c() and seq() to building a simple data frame with data.frame()?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.

Before You Leave This Page

  • Confirm that chapter2_practice.R is saved in your course folder.
  • Confirm that you can run code from the Source pane into the Console.
  • Confirm that your script includes comments or memos where the chapter asks for them.
  • Confirm that your vectors and spectrum_shift data frame are created correctly.
  • Run the script from top to bottom to catch order-of-execution problems.
