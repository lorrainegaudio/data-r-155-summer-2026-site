Purpose

Structures matter because the way values are organized in R affects how you label them, access them, and interpret them later.

In this guided activity, you will build and work with factor vectors and named vectors while continuing to strengthen your R Notebook workflow.

That work is worth doing now because later analysis depends on understanding how R stores categories, labels values, and retrieves specific pieces of an object. These skills will help you create clearer objects, index them more intentionally, and document your work transparently.

Chapter

  • Chapter_4_🏗️Structures.html
  • Focus on R Notebooks—Part 2, especially editing the YAML header, working with code chunks, and writing transparent memos.
  • Then pay close attention to the sections on factor(), levels and ordering, named vectors with names(), and bracket indexing by position or by name.

Data

No external dataset required.

This guided activity works with small vectors you create in your notebook, including factor examples such as grades and sizes and a named vector such as nerdy_palette.

Those examples are used to practice categorical data, levels, naming, and indexing without needing to import a separate data file.

Guiding Questions

  • How does the chapter connect transparency in memo writing to reproducible work in an R Notebook?
  • When a character vector becomes a factor, what changes, and why do levels matter for later analysis?
  • What is the difference between an object name and the element names inside a named vector?
  • When should you index a vector by position, and when is indexing by name the better choice?
  • As you follow the examples, what checks help you tell whether R is showing the class, the stored type, or the values you expected?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.
