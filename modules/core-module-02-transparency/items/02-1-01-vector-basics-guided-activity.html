Purpose

Vector basics matter because vectors are one of the main ways R stores values, performs calculations, and supports later data analysis.

In this guided activity, you will learn how to work in an R Notebook, run code in chunks, memo your thinking clearly, and build, inspect, and combine vectors in several useful ways.

That work is worth doing now because later filtering, subsetting, recoding, and dataset work all depend on understanding what kind of object you created and how R is treating it. These skills will help you build cleaner notebooks, diagnose simple object-type problems earlier, and use vectors more intentionally in R.

Chapter

  • Chapter_3_📏Vector.html
  • Focus on the overview, R Notebooks—Part 1, Memoing in R Notebooks, and the sections on vector arithmetic, recycling, class(), character strings with paste(), and logical vectors.
  • Pay close attention to how the chapter connects object checks and memoing to reproducible work.

Data

No external dataset required.

This guided activity uses small numeric, character, and logical vectors created in the notebook so you can focus on vector arithmetic, recycling, object checks, string combination, and memoing without needing to import a separate data file.

Guiding Questions

  • How does the chapter show the difference between using an R Script and using an R Notebook for memoing and reproducible work?
  • What does vectorization mean in the arithmetic examples, and what changes when recycling happens because vectors have different lengths?
  • When you check an object with class() or related structure checks, what are you trying to confirm before you keep working?
  • How does paste() combine strings, and what do you notice when it works across vectors element-by-element?
  • Why are logical vectors worth learning now, even before you start formally filtering rows in datasets?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.
