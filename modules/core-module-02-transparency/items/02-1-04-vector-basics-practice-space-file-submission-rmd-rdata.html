Purpose

This assignment collects your Chapter 3 Vectors Practice Space work as an R Notebook file and a workspace file.

For this chapter, those uploaded files matter because they show how you created vectors, checked their structure and length, used vector arithmetic, and documented your work in a reproducible notebook workflow that can be rerun and reviewed later.

File Upload

  • Required files to submit: chapter3_practice.Rmd, chapter3_practice.RData
  • File types: .Rmd, .RData
  • These files should show:
    • your completed Chapter 3 Practice Space work for Task 1 through Task 5
    • the actual R code used in your R Notebook
    • your vector-building work with sales_q1 and sales_q2
    • your vector-arithmetic work with growth and anomaly
    • your exponent work with exponents and binary_powers
    • your supporting vectors cursed_seq and travel_jumps
    • your checks after code, such as print() and length()
    • your written comments or memo notes for each task, including any tenacity notes if you had to troubleshoot an error
  • Based on: Chapter 3: Vectors
  • If applicable, include: a YAML header with the title Chapter 3 Practice, task headings, and a saved workspace that includes the main vectors you created in the Practice Space.

How You Will Be Marked

✅ Complete if all are true

  1. Code + tasks
    • The uploaded chapter3_practice.Rmd contains the completed Task 1 through Task 5 for Chapter 3.
    • The work is clearly for Chapter 3 vectors, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The notebook shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific vector work: building sales_q1 and sales_q2, creating growth, building binary_powers, and computing anomaly.
    • The notebook includes the supporting chapter objects exponents, cursed_seq, and travel_jumps.
  2. Memos
    • The file includes the required written comments or memo responses for each task.
    • The task comments explain what the output shows, not just what code was typed.
    • If you encountered an error, the notebook documents your tenacity by briefly explaining how you troubleshot it.
    • If outside help was used, that help is briefly disclosed in a memo or note.
  3. Reproducibility / chapter-specific requirements
    • The submission includes the correct files: chapter3_practice.Rmd and chapter3_practice.RData.
    • The notebook starts from a clean Environment and runs top-to-bottom in order.
    • The notebook includes the Chapter 3 YAML title and keeps the work in a single reproducible document.
    • The .RData file contains the main Practice Space objects: sales_q1, sales_q2, growth, exponents, binary_powers, cursed_seq, travel_jumps, and anomaly.
    • The notebook includes checks that confirm vector length or printed output where the tasks ask for them.
    • The files document the Chapter 3 workflow clearly enough that the vector work can be reviewed and rerun later.

🟡 Not Ready if any are true

  • No attachment.
  • Wrong artifact.
  • Only one required file type is submitted instead of both.
  • The file cannot be opened.
  • The notebook is blank, incomplete, or unrelated to Chapter 3.
  • The required vectors, checks, or memo notes are missing.
  • The notebook does not run from top to bottom from a clean Environment.

If your submission is marked 🟡 Not Ready, revise to address the missing elements and resubmit for reevaluation before the assignment locks.

❌ Missing

No submission.

Before You Submit

  • Confirm your notebook is named chapter3_practice.Rmd.
  • Confirm your workspace is named chapter3_practice.RData.
  • Confirm both files are uploaded.
  • Confirm the notebook includes Tasks 1–5.
  • Confirm your YAML title says Chapter 3 Practice.
  • Confirm the notebook runs from a clean Environment.
  • Confirm your memo notes explain the output and any troubleshooting.
