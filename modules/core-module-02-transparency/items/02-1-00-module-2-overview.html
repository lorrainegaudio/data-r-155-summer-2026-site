✍️ Transparency

This module focuses on transparency: making your R work clear enough that someone else can see what you did, why you did it, and how your output was created. You will move from basic scripts into R Notebooks, where code, output, and memo notes can live in the same document. You will also begin working more seriously with vectors, including numeric, character, and logical vectors.

The Core work focuses on vector basics and R object structures. You will practice vector arithmetic, recycling, object checks with functions such as class() and length(), string combination with paste(), factors with factor(), named vectors with names(), and indexing by position, name, or logical condition. You will also keep strengthening your notebook workflow by writing clear headings, running code chunks, memoing your thinking, and checking that your work runs from a clean Environment.

The module pattern should feel familiar from Module 1: complete the Guided Activity, use supplemental materials when you need another explanation, complete the Practice Space, and submit the required file. The difference is that your work now needs to be more transparent. You are not just getting the code to run; you are also showing the reasoning, checks, and notes that make your work easier to review and reproduce.

Learning Objectives

By the end of this module, the successful student will be able to:

  • Create and edit an R Notebook with a usable YAML header, code chunks, section headings, and memo text.
  • Build numeric, character, and logical vectors directly in R.
  • Use vector arithmetic to combine vectors and interpret the resulting output.
  • Recognize recycling when vectors of different lengths are used together in arithmetic.
  • Apply object checks such as class(), length(), mode(), and str() to inspect how R is treating stored values.
  • Use paste() to combine character strings and observe how it works across vectors.
  • Create and inspect factor vectors with factor(), including levels and ordering.
  • Create named vectors with names() and retrieve values by name or by position.
  • Use logical filtering and bracket indexing to select values that meet a condition.
  • Explain coercion when mixed values in one vector are converted to a shared data type.
  • Document code output, troubleshooting, and outside help clearly in notebook memo notes.

Why This Topic Is Important

Vectors are one of the basic building blocks of R. If you do not understand how R stores and combines vector values, later data work will feel random. This module helps you see what kind of object you created, how long it is, what type of values it contains, and how R behaves when you combine or compare those values.

Transparency matters because code that “worked once” is not enough. In real analytical work, you need to leave a trail: what you ran, what output appeared, what you checked, and what you concluded. R Notebooks help you keep code, output, and written explanation together, which makes your work easier to review, revise, and troubleshoot.

This module also prevents common beginner mistakes: mixing data types without noticing coercion, confusing object names with element names, indexing the wrong position, forgetting that R is case sensitive, and moving ahead without checking output. These habits prepare you for later work with filtering, subsetting, recoding, and larger datasets.

Strategies for Success

  • Run notebook chunks as you write them. Do not wait until the end to discover that earlier code failed.
  • Check your objects often with class(), length(), mode(), or str() before using them in later code.
  • Watch for coercion. If a vector mixes numbers and text, R may convert the values to a shared type.
  • Keep object names and element names separate in your thinking. A vector can have one object name and many named elements inside it.
  • Use bracket indexing carefully. Indexing by position, name, and logical condition are related, but they are not interchangeable.
  • When using logical filters, print the result and ask whether the selected values match the condition you intended.
  • Write memo notes while you work, not after you forget what happened.
  • Run the notebook from a clean Environment before submitting so you know the document can reproduce the work in order.
