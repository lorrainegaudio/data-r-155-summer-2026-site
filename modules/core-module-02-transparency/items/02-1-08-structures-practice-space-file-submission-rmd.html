Purpose

This assignment collects your Chapter 4 Practice Space work as an R Notebook file.

For this chapter, the uploaded file matters because it shows how you worked with factors, named vectors, positional indexing, and logical filtering while documenting your reasoning, troubleshooting, and memoing in one reproducible notebook.

File Upload

  • Required file to submit: chapter4_practice.Rmd
  • File type: .Rmd
  • This file should show:
    • your completed Chapter 4 Practice Space work for Task 1 through Task 5
    • the actual R code used in the notebook
    • your named-vector work with nerdy_palette
    • your positional indexing work using the 2nd, 4th, and 6th elements of nerdy_palette
    • your logical filtering and indexing work with change, including positive_change and last_two
    • your one-sentence reflection explaining why numeric values may be coerced to character when mixed in a single vector
    • your memo notes about learning, challenges, troubleshooting, and any resources used
  • Based on: Chapter 4: Structures
  • If applicable, include: a clear YAML header, headings for each task, and brief disclosure of outside help used while troubleshooting.

How You Will Be Marked

✅ Complete if all are true

  1. Code + tasks
    • The uploaded chapter4_practice.Rmd contains the completed Task 1 through Task 5 for Chapter 4.
    • The work is clearly for Chapter 4 factors, indexing, and named vectors, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The notebook shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific indexing work with nerdy_palette by name and by position.
    • The submission includes the logical filtering and indexing task with change, including positive_change and last_two.
    • The file includes the required reflection on coercion in mixed vectors.
  2. Memos
    • The file includes memo responses for each task where memoing is requested.
    • The memoing explains what you learned, checked, or noticed, not just what code you typed.
    • If you encountered errors, the notebook documents your tenacity by explaining how you troubleshot the problem.
    • If you used outside help, the notebook briefly identifies the resource and what kind of help it provided.
    • The memoing reflects the chapter’s expectation for transparency in your workflow.
  3. Reproducibility / chapter-specific requirements
    • The submission is the correct file format: chapter4_practice.Rmd.
    • The notebook starts from a clean Environment and runs top-to-bottom in order.
    • The file includes a usable R Notebook structure with code and memo text in the same document.
    • The latest edits are saved in the submitted notebook.
    • The file documents the chapter workflow clearly enough that the vector and indexing work can be followed and checked later.

🟡 Not Ready if any are true

  • No attachment.
  • Wrong artifact.
  • The file cannot be opened.
  • The notebook is blank, incomplete, or unrelated to Chapter 4.
  • The required named indexing, positional indexing, logical filtering, or coercion reflection is missing.
  • The required memo notes or transparency notes are missing.
  • The notebook does not run from top to bottom from a clean Environment.

If your submission is marked 🟡 Not Ready, revise to address the missing elements and resubmit for reevaluation before the assignment locks.

❌ Missing

No submission.
