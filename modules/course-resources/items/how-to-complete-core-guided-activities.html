Purpose

This page explains what Guided Activities are, how to complete them, and where to get help.

Each Topic Guided Activity is your primary learn-by-doing work. It is designed to help you build real fluency in R by reading, running the code yourself, and documenting your process so you can reproduce it later.

Guided Activities directly prepare you for the Practice Space. If you skim the chapter or watch only the provided video, you will miss required steps.

What to Expect in a 🌱 Core Guided Activity

Each Guided Activity is built into the Chapter HTML and may include:

  • R coding examples you are expected to run in RStudio.
  • Instructions for creating or editing specific files, such as R scripts (.R), R Notebook files (.Rmd), or Quarto files (.qmd).
  • Memo examples and prompts, which are short notes you write to document learning, confusion, errors, and fixes.
  • Guidance on how to use and how not to use AI tools in this course.
  • Sometimes, a walkthrough video. Videos are helpful, but they are not complete.
  • Sometimes, a dataset download linked on the same page as the chapter.

The Chapter HTML is required. Videos are support materials and do not include all required content.

How to Complete a 🌱 Core Guided Activity

  1. Open the Chapter HTML.
    Open it from the module page. If Canvas is unavailable, use the local course site version.
  2. Open RStudio Desktop and work alongside the chapter.
    When the chapter tells you to create or open a file, do it immediately.
  3. Type the code yourself and run it.
    Do not just read code blocks. Do not copy and paste unless directed. Guided Activities are designed to build muscle memory: typing, running, checking output, and fixing mistakes.
  4. Pause and verify your output matches the chapter.
    If your output differs, do not plow ahead. Troubleshoot first:
    • Re-run lines in smaller chunks.
    • Check spelling, punctuation, parentheses, and commas.
    • Confirm object names match exactly.
  5. Write memos when prompted and when you get stuck.
    Memos are part of learning R responsibly. Use them to document:
    • what you were trying to do
    • what happened, including output or error messages
    • what you changed to fix it and what resource helped
  6. If the Guided Activity includes a dataset, download and place it correctly.
    Use the dataset link on the Guided Activity page and save it exactly where the chapter specifies, often inside a data/ folder. If the chapter tells you to verify the file loads, do that before moving on.
  7. Use videos as support, not as the main content.
    Watch the video when you need a walkthrough or a second explanation. Then return to the HTML to complete the remaining required steps.
  8. Finish by checking that you are ready for the Core Practice Space.
    Before leaving the Guided Activity, confirm:
    • your file is saved
    • required objects exist in your Environment, if the chapter calls for it
    • you can run the workflow again from a clean start, if instructed

Getting Help

If something still feels unclear during or after completing the Guided Activity:

  • Ask peers in your discussion group.
  • Message the instructor through Canvas Inbox when Canvas is available.
  • Use office hours or book a meeting with the instructor.
  • You may use AI tools during Guided Activities, but use them as learning support, not as a replacement for doing the work yourself.
  • Use the Supplemental Readings and Multimedia page for alternative explanations. Supplemental materials are optional and do not match the chapter 1:1.

Code Language

In most instructional materials, code will be visually marked so you can distinguish code from instructions. For example, you may see function names such as mean(), paste(), or head().

When you see code, type it carefully in RStudio and run it. When you see instructions, read them before running the next line of code.

For information on using R and screen readers, see Resources for Using R with Screen Readers.
