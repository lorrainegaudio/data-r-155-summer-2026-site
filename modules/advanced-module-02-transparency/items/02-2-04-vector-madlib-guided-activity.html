Purpose

In this Advanced Guided Activity, you will use vectors to generate two randomized MadLib stories in R.

You will build numeric, integer, logical, and character vectors, convert one existing character vector into factor class, and run story code that depends on the objects you created.

This activity builds directly on recent Core work with vectors, logical operators, R Notebooks, and factor class. It matters in a realistic R workflow because small naming, length, or class mistakes can break later code, so you will practice building objects carefully and checking your work before using those objects in a larger script.

Advanced Guided Activity

  • Focus: vector construction, factor conversion, randomized story output, and reproducible notebook workflow
  • Time estimate: 1 hour
  • Do: Friday
  • Open: 🤪 Vector MadLib
  • Use: RStudio, the Vector MadLib R Notebook template, the activity HTML, and your course notes on vectors, logical operators, and factors
  • Have ready: RStudio, your course folder, and the recent Core material on vector types, length(), class(), logical operators, and factor class
  • What you will do: Complete the provided R Notebook template, run the story code, and copy your final MadLib story output into a text file.
  • Produce:
    • a completed working R Notebook for your own use while coding,
    • printed output for Story 1 and Story 2, and
    • a plain text submission that contains your name, both story outputs, and a short transparency memo.

Activity File

  • 🤪 Vector MadLib
  • Use this file as the activity guide. Pay close attention to the object names, required vector lengths, the factor conversion step, and the Story section.

R Notebook Template

Download the Vector MadLib R Notebook template

  1. Download the R Notebook template.
  2. Move it into your course folder.
  3. Open it in RStudio.
  4. Replace YOUR NAME in the YAML header with your name.
  5. Work through the notebook from top to bottom.

The template contains starter chunks and spaces for your comments and memos. Fill in the blanks, run each chunk, and do not rename the required objects.

Guiding Questions

  • How do exact object names affect whether later code can find and use your vectors?
  • How do length() and class() help you check that an object is ready to use?
  • What is the difference between numeric, integer, logical, and character vectors in this activity?
  • What changes when you convert one character vector into a factor object called story_factor?
  • If the story code errors, what does the error suggest: a missing object, a misspelled object name, a length issue, or a quotation mark problem?

You do not submit separate answers to these questions unless the activity tells you to. Use them to focus your reading, coding, memo notes, and interpretation as you work through this part.

Before You Move On

  • Check that all required vectors exist and have the expected length.
  • Check that your object names match the activity exactly.
  • Check that story_factor is a factor, not still just a character vector.
  • Run the story code and confirm that both Story 1 and Story 2 print successfully.
  • Copy your final story output into the submission format requested on the next page.
