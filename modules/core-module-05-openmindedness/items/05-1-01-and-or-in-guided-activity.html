Purpose

Logical filtering is how analysts turn a plain-English rule into a precise subset of data. In this guided activity, you will practice base R subsetting with [ ], build conditions with &, |, %in%, and !, and learn why parentheses matter when conditions get more complex.

That work is worth doing now because later analysis depends on keeping the right rows for the right reason, not just writing code that runs. These skills will help you build cleaner subsets, catch silent logic errors, and explain your filtering choices clearly in R.

Data

No external dataset required: Built-in mtcars dataset used during the guided activity.

This chapter’s guided activity uses mtcars to practice bracket subsetting, negative indexing, logical tests, and multi-condition row filters in base R.

The gapminder dataset appears later in the Practice Space, but it is not the main dataset for this Guided Activity page.

Guiding Questions

  • How does the chapter distinguish between excluding rows or columns by position and filtering rows by a logical condition?
  • When should you use &, when should you use |, and when is %in% the cleaner choice?
  • What changes when you put ! in front of a logical test, and how is that different from using !=?
  • Why do grouping parentheses matter when a filter mixes & and | in the same statement?
  • As you follow the examples, what quick checks help you confirm that your subset really matches the rule you meant to write?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.

Before You Leave This Page

  • Run the examples yourself instead of only reading them.
  • Use checks after each subset to confirm your code matches the rule you intended.
  • Memo any filtering mistake you catch, especially if the code ran but returned the wrong rows.
