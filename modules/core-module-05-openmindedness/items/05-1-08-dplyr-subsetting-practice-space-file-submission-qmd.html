Purpose

This assignment collects your Chapter 11 Practice Space work as a Quarto .qmd file. For this chapter, the uploaded file matters because it shows how you used select() and filter() to build focused subsets, check that those subsets are correct, and document your reasoning in a reproducible workflow file that can be run again from top to bottom.

File Upload

  • Required file to submit: chapter11_practice.qmd
  • File type: .qmd
  • This file should show:
    • your completed Chapter 11 Practice Space work for Task 0 through Task 7,
    • the actual R code used for select() and filter() tasks,
    • the required chapter objects: C_Overload, star_names, No_Florida, Fast_n_Furious, and filtered_cars,
    • your checks after code, such as head(), unique(), nrow(), range(), or all() where used in the tasks,
    • your memo responses written as normal text outside code chunks, including the Chapter 11 reflection, and
    • brief disclosure of outside help in a memo if you used it.
  • Based on: Chapter 11: dplyr Subsetting
  • If applicable, include: headings for Task 0 through Task 7, code that loads dplyr and dslabs, and Quarto chunk controls with #| if you used them.

How you will be marked

✅ Complete if all are true:

  1. Code + tasks

    • The uploaded chapter11_practice.qmd contains the completed Task 0 through Task 7 for Chapter 11.
    • The work is clearly for Chapter 11 dplyr subsetting, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The file shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific subsetting work using select() and filter().
    • The file includes the required chapter objects: C_Overload, star_names, No_Florida, Fast_n_Furious, and filtered_cars.
    • The submission includes the Chapter 11 reflection comparing dplyr subsetting with base-R brackets [ ].

  2. Memos

    • The file includes memo responses for each required task or step where memoing is requested.
    • Each task includes a goal memo in the student’s own words.
    • The memos answer the chapter’s specific comment questions, including what select() did, whether star_names stays a one-column data frame, what rows were removed from murders, what each filtering condition means for Fast_n_Furious, and what all() verifies for filtered_cars.
    • The reflection memo explains the utility of select() and filter() as compared with base-R bracket subsetting.
    • If outside help was used, that help is briefly disclosed in a memo.

  3. Reproducibility / chapter-specific requirements

    • The submission is the correct output format: chapter11_practice.qmd, not a different file type.
    • The file runs from top to bottom without errors when executed in order.
    • The document includes code that loads the required packages dplyr and dslabs.
    • The file is organized with headings for Task 0 through Task 7.
    • Memo responses are written as normal text outside code chunks.
    • The chapter objects have the expected content:
      • C_Overload keeps only year and carbon_emissions.
      • star_names has one column named star.
      • No_Florida has no rows with state == "Florida".
      • Fast_n_Furious keeps only manual, 8-cylinder cars with qsec < 15.
      • filtered_cars keeps rows where disp > 250 or disp < 160 and only the first three columns.
