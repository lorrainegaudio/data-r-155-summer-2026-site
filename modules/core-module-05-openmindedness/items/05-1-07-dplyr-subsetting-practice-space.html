Purpose

This Practice Space checks whether you can apply the main subsetting skills from Chapter 11: dplyr Subsetting after completing the associated Guided Activity. In this chapter, you use select() to keep specific columns, filter() to keep specific rows, and Quarto to organize code and memo responses in a reproducible workspace.

This practice matters because subsetting is one of the most common tasks in data analysis. You need these skills before you can summarize, graph, or model data later in the course.

What this Practice Space Checks

  • Use select() to keep specific columns from a data frame.
  • Use filter() to keep or remove rows based on one or more conditions.
  • Apply piping with %>% to build readable subsetting workflows.
  • Distinguish between changing columns with select() and changing rows with filter().
  • Interpret verification checks such as head(), unique(), nrow(), and all() to confirm that a subset is correct.

Task 3 Part B

Row Filtering

Pick the best multiple-choice answer that explains why checking the unique values of state before and after filtering is useful.

# ⚡ Completed code pattern

Task 5

OR Logic + Column Slice

🎯 From 🚗 mtcars, keep only the cars whose displacement is either under 160 or over 250, and then show only the first three columns of those rows. Store as filtered_cars.

# ✅ Pre-check

range(mtcars$disp)

🗣 This tells the spread of disp in mtcars so you can see whether the cutoffs 160 and 250 make sense.

# ⚡ Type this in a Quarto code chunk. Fill in the blanks.

filtered_cars <- mtcars %>%
  ________(disp > 250 ________ disp < 160) %>%
  ________(1:3)
# ✅ Verify

all(filtered_cars$disp > 250 | filtered_cars$disp < 160)

Answer format: fill in the row-filtering function, the OR operator, and the column-selection function.

Task 6

Reflection

📝 Which statement best compares the utility of dplyr verbs select() and filter() to base-R brackets [ ] for subsetting?

  • dplyr always produces vectors instead of data frames, whereas brackets cannot subset data frames.
  • Bracket indexing cannot subset rows or columns and is only useful when making plots.
  • dplyr verbs make it easier to tell whether you are changing rows or columns, while base-R brackets are still useful for direct row-and-column indexing.
