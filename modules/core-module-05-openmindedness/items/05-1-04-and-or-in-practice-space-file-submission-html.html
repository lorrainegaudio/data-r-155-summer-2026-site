Purpose

This assignment collects your Chapter 10 Practice Space work as a rendered HTML report. For this chapter, the uploaded file matters because it shows how you used base R bracket subsetting with &, |, %in%, and related logical tests to narrow the gapminder data into a defensible NGO health-pilot shortlist and document that process in a reproducible report.

File Upload

  • Required file to submit: chapter10_practice.html
  • File type: .html
  • This file should show:
    • your completed Chapter 10 Core Practice Space work from setup through the Core filtering tasks,
    • the actual R code used to work with dslabs::gapminder,
    • your stored objects and workflow, including target_year, region_1, region_2, region_3, focus_pool, brief_sheet, and needs_screen,
    • your checks after code, such as colnames(), unique(), nrow(), names(), range(), and sum() where the tasks ask for them,
    • your short written memo responses to the chapter’s prompt questions, and
    • a complete rendered report that can be opened and reviewed outside RStudio.
  • Based on: Chapter 10: &, |, and %in%
  • If applicable, include: headings for each task, your YAML with title/author/date, and only the rendered HTML file for submission, not the .Rmd source file.

How You Will Be Marked

✅ Complete if all are true:

  1. Code + tasks
    • The uploaded chapter10_practice.html contains the completed Core Practice Space work for Chapter 10.
    • The work is clearly for Chapter 10 logical filtering with bracket subsetting, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The report shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific workflow objects, including target_year, the three stored regions, focus_pool, brief_sheet, and needs_screen.
    • The code shows the chapter’s main logical-filtering work with equality tests, relational operators, &, and %in% inside bracket subsetting.
    • The report includes the indicator inspection work for life_expectancy, infant_mortality, and fertility before creating the final screening filter.
  2. Memos
    • The file includes written responses for each required prompt in the Core tasks.
    • The memo responses explain why you selected your target regions.
    • The file includes your explanation of what %in% and & are doing in the focus_pool filter.
    • The file includes your explanation of how negative indices work for dropping columns by position.
    • The file explains how the range and count checks helped you choose reasonable cutoff values for a, b, and c.
    • The file explains how the three screening conditions combine to create needs_screen.
  3. Reproducibility / chapter-specific requirements
    • The submission is the correct output format: chapter10_practice.html, not the .Rmd source file.
    • The HTML report rendered successfully and opens as a complete report.
    • The code includes the required data setup with library(dslabs) and gapminder <- dslabs::gapminder.
    • The file does not leave install.packages("dslabs") uncommented in a way that would break knitting.
    • The report includes the task-by-task workflow in order so the subsetting process can be followed and rerun.
    • The report includes enough checks to verify that the filtering and screening steps worked as intended.
