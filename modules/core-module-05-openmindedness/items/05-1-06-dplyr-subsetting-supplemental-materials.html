Purpose

Review the supplemental resources on this page if you would like an alternative explanation, example, or walkthrough for concepts in this chapter or need extra help with a specific skill.

These materials are optional and do not replace the chapter. Use them when you need another explanation of Quarto, dplyr, select(), filter(), the pipe, or row-and-column subsetting.

Resources

Quarto Workflow

  • Quarto for RStudio
    • This reading is most useful for learning the basic Quarto workflow in RStudio: create a .qmd file, write YAML, add code chunks and Markdown text, then render the document to HTML or another output format. It also helps clarify that Quarto is the next-generation version of R Markdown and shows where visual editing, source editing, and chunk execution fit into the workflow.
  • Quarto Cheatsheet
    • Use this cheatsheet as a quick reference for the Quarto syntax and workflow pieces you need while writing reports, especially code cells, execution options, inline code, figures, and tables. It is the page to keep open when you need a fast reminder of source-file structure or common authoring commands without reading long documentation.

dplyr Reference Guides

  • dplyr::select() Reference Guide
    • This guide is most useful for learning exactly how select() keeps, drops, and reorders columns by name. Look for helpers like :, starts_with(), ends_with(), contains(), and everything(), since those tools make column selection faster than base R indexing.
  • dplyr::filter() Reference Guide
    • This guide is most useful for understanding how filter() keeps rows only when conditions evaluate to TRUE. It is especially helpful for checking the exact behavior of multiple conditions, %in%, grouped filtering, and the important rule that rows with NA in the condition are dropped.

Subsetting with filter(), select(), and the Pipe

  • Data Science: A First Introduction
    • 1.7 Creating subsets of data frames with filter and select
      • This reading is most useful for a first introduction to the difference between rows and columns when subsetting a data frame. It helps you see the basic workflow of using filter() for rows and select() for columns before combining those verbs in longer pipelines.
    • 3.5 Using select to extract a range of columns
      • Use this reading when you need clearer examples of choosing columns with select(), especially by name, position-like ranges with :, and tidyselect helpers. It is especially useful for understanding that select() is about keeping or dropping variables, not filtering observations.
    • 3.6 Using filter to extract rows
      • This reading is most useful for learning how logical conditions drive row filtering with filter(). Focus on the comparisons and operators it demonstrates, including ==, !=, >, <, &, |, and %in%, because those are the core tools for keeping only the rows you want.
  • Data Analysis and Visualization in R for Ecologists
    • Manipulating data
      • This reading is most useful for seeing how dplyr verbs work on a real dataset, especially when you are moving from base R indexing into cleaner data-manipulation workflows. It helps clarify the practical role of verbs like select(), filter(), mutate(), and group_by() in day-to-day wrangling.
    • The Pipe
      • This reading is most useful for understanding what the pipe does and why it makes multi-step data work easier to read. Watch for how the pipe passes the result of one step into the next so you can read a wrangling workflow from left to right instead of nesting functions inside each other.
  • Data Analysis and Visualization with R for Genomics
    • Data Wrangling and Analytics with Tidyverse
      • This reading is most useful for seeing a broader tidyverse wrangling workflow in one place, including select(), filter(), mutate(), group_by(), and summary operations. It is especially helpful when you need to see how these verbs fit together as a sequence rather than as isolated commands.
