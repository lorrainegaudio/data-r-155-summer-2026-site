Purpose

Review the supplemental resources on this page if you would like an alternative explanation, example, or walkthrough for concepts in this chapter, or if you need extra help with a specific skill.

These materials are optional. They do not replace the Chapter 12 guided activity.

Resources

  • Quarto: HTML Basics
    • This reading is most useful for learning the basic HTML output settings you can control in a Quarto document, especially options like a table of contents, section numbering, and other document-level features. Use it when you want your rendered report to be easier to navigate and read, not just able to run code.
  • Data Science: A First Introduction
    • 3.7 Using mutate to modify or add columns
      • This reading is most useful for learning how mutate() creates a new column or changes an existing one inside a data frame. Watch for the basic workflow of defining a new variable from existing columns, because that is the core move behind many later wrangling tasks.
    • 3.10 Apply functions across many columns with mutate and across
      • Use this reading when you need to do the same transformation to multiple columns at once instead of writing one line per variable. It is especially helpful for understanding how across() works inside mutate() to apply one function across a selected set of columns.
    • 3.11 Apply functions across columns within one row with rowwise and mutate
      • This reading is most useful for clarifying the difference between working down columns and working across values within one row. Look for how rowwise() changes the workflow so mutate() can compute a row-based result, such as combining several columns for each case.
  • Data Analysis and Visualization in R for Ecologists: Making new columns with mutate()
    • This reading is most useful for a practical introduction to using mutate() on a real dataset to add derived variables. It shows that mutate() is not just abstract syntax; it is a way to build new columns directly from existing measurements in a reproducible workflow.
  • R for Social Scientists: Introducing dplyr and tidyr (dplyr section)
    • This reading is most useful for understanding the broader dplyr verb workflow, especially how verbs like select(), filter(), mutate(), and summarize() fit together. Use it when you need to see mutate() as one part of a larger tidy data workflow rather than as an isolated command.
  • Data Analysis and Visualization with R for Genomics: Data Wrangling and Analytics with Tidyverse (start at Mutate and finish the page)
    • This reading is most useful for reinforcing how mutate() works in a longer wrangling sequence and how it connects to later steps like grouping and summarizing. It is especially helpful when you need more practice seeing column creation, transformation, and tidyverse chaining all in one place.

How to Use These Materials

  • Start with the Chapter 12 guided activity before using these resources.
  • Use the Quarto resource if your rendered HTML report is hard to read or navigate.
  • Use the mutate() readings if you are unsure whether you are creating a new column or changing an existing one.
  • Use the across() and rowwise() readings only after you are comfortable with basic mutate().
  • Use the video when you need to see several dplyr verbs working together in sequence.
