Purpose

Review the supplemental resources on this page if you would like an alternative explanation, example, or walkthrough for concepts in this chapter, or if you need extra help with a specific skill.

These materials are optional. They do not replace the Chapter 13 guided activity.

Resources

  • Data Science: A First Introduction: 3.9 Aggregate data with summarize and map
    • This reading is most useful for learning how summarize() turns many values into a smaller set of summary results, such as a mean, count, or other aggregate. Look closely at the difference between summarizing a whole column and summarizing groups of rows, because that is the key step before using group_by() and summarize() together.
  • Data Analysis and Visualization in R for Ecologists: The split-apply-combine approach
    • This reading is most useful for understanding the logic behind grouped summaries: first split the data into groups, then apply a function to each group, and finally combine the results into one table. It helps you see why group_by() and summarize() belong together and what problem that workflow is solving.
  • Data Science: A First Introduction: 3.8 Combining functions using the pipe operator
    • This reading is most useful for showing how the pipe operator connects multiple wrangling steps into one readable workflow. Focus on how the pipe lets you read code from left to right, which makes combinations like filtering, selecting, grouping, and summarizing easier to follow than nested functions.

None at this time.

How to Use These Materials

  • Start with the Chapter 13 guided activity before using these resources.
  • Use the summarize() reading if you are unsure how one column becomes one summary value.
  • Use the split-apply-combine reading if group_by() feels abstract.
  • Use the pipe reading if your grouped summary code is hard to read from left to right.
  • Return to your Quarto notes and test the ideas with the same data used in the chapter.
