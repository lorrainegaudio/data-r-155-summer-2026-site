Purpose

This Practice Space checks whether you can apply the main recoding skills from Chapter 12: Recoding after completing the associated Guided Activity.

In this chapter, you use ifelse(), case_when(), and mutate() to label values, build cleaner variables, and document your work in Quarto with checks and memos.

This practice matters because recoding is a common data-analysis task, and this course expects you to produce work that can be re-run, checked, and understood later.

What This Practice Space Checks

  • Use vectorized conditionals such as ifelse() and case_when() to return one result for every position.
  • Apply mutate() to create new columns or overwrite an existing column in a data frame.
  • Distinguish between recoding with a new output variable and updating an existing variable with transformed values.
  • Interpret checks such as table(), head(), summary(), and colnames() to confirm that recoding worked as expected.
  • Document a reproducible Quarto workflow with headings, memoing, inline R code, rendering, and brief AI disclosure when outside help was used.

Task 3: 📊 Recode with mutate() and case_when()

🎯 Create mpg_class with three labels using case_when(): "High MPG" if mpg is greater than 25; "Low MPG" if mpg is less than 15; otherwise "Medium MPG".

# ⚡ Code
mpg_recode <- mtcars %>%
  ________(mpg_class = ________(
    mpg > 25 ~ ________,
    mpg < 15 ~ ________,
    TRUE     ~ ________
  ))
# ✅ Checks
table(mpg_recode$mpg_class)  # expect 3 levels

Task 4: ↔️ Recode: Name Focus

🎯 With the dataset stars from the dslabs package, classify each star by temperature range in a new column called temp_group. Store the result as stars_recode.

Use these rules:

  • Temperatures less than 4000 are "Cool".
  • Temperatures between 4000 and 7500 are "Warm".
  • All stars more than 7500 are "Hot".
# ✅ pre-check
?dslabs::stars
stars <- dslabs::stars

# View(stars)
colnames(stars)
summary(stars$temp)

🗣 Which column contains temperature values in the data frame stars? Identify where the new output column name goes: left or right of the =?

# ⚡ Code
________ <- ________ %>%
  mutate(
    ________ = case_when(
      ________ < 4000 ~ ________,
      ________ <= 7500 ~ ________,
      ________ ~ ________
    )
  )
# ✅ Checks
table(stars_recode$temp_group)  # expect 3 levels

🗣 Comment on the output. Do you see the expected labels? How many "Cool", "Warm", and "Hot" stars are there?
