Purpose

This assignment collects your Chapter 12 Practice Space work as a rendered HTML report.

For this chapter, the uploaded file matters because it shows how you used ifelse(), case_when(), and mutate() to recode values, create new columns, overwrite an existing column, and document those choices in a reproducible Quarto workflow.

File Upload

  • Required file to submit: chapter12_practice.html
  • File type: .html
  • This file should show:
    • your completed Chapter 12 Practice Space work on vectorized conditionals and mutate()
    • the actual R code used for Task 1 through Task 5
    • your recoding work products, including height_label, mpg_recode with mpg_class, stars_recode with temp_group, and international_friendly_heights
    • your checks after code, such as table(), head(), or other fitting checks
    • your memo responses to the chapter’s comment questions
    • any brief disclosure of outside help used while completing the work
  • Based on: Chapter 12: Recoding
  • If applicable, include: headings, section numbering, a table of contents, code that loads dplyr and dslabs, memoing with the 🎯/⚡/✅/🗣 structure, and inline R code where used in your memoing workflow.

How You Will Be Marked

✅ Complete if all are true

  1. Code + tasks
    • The uploaded chapter12_practice.html contains the completed Task 1 through Task 5 for Chapter 12.
    • The work is clearly for Chapter 12 recoding, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The report shows the actual R code used for the chapter tasks.
    • The submission includes the chapter-specific recoding work, including ifelse(), case_when(), and mutate().
    • The file includes the required chapter-specific outputs: height_label, mpg_recode with mpg_class, stars_recode with temp_group, and international_friendly_heights.
    • The report includes the concept-check task and the coding tasks on simple ifelse(), recoding with mutate() and case_when(), temperature grouping in stars, and unit conversion by overwriting height.
  2. Memos
    • The file includes memos for each required task or step where memoing is requested.
    • Each coding task includes a goal memo before the code and a short memo after the code that explains what you learned, confirmed, or discovered.
    • The file answers the chapter’s specific comment questions, including questions about which column is being used, where the new output column name goes, whether the expected labels appear, and whether the values look correct after recoding or conversion.
    • The memos explain what the recoding or conversion shows, not just what code was run.
    • If outside help was used, that help is briefly disclosed in a memo.
  3. Reproducibility / chapter-specific requirements
    • The submission is the correct output format: chapter12_practice.html, not the .qmd source file.
    • The HTML report rendered successfully and displays as a complete report.
    • The report includes code that loads the required packages dplyr and dslabs.
    • The report includes headings, section numbering, and a table of contents as required for the Chapter 12 Quarto setup.
    • The file includes needed checks after code, such as table() and head(), so the recoding results can be verified.
    • The file documents the chapter workflow clearly enough that the recoding and column-building work can be reviewed and rerun later.

🟡 Not Ready if any are true

  • No attachment.
  • Wrong artifact.
  • The file cannot be opened.
  • The uploaded file is the .qmd source file instead of the rendered .html report.
  • The report is blank, incomplete, cut off, or unrelated to Chapter 12.
  • The required tasks, code, checks, or memos are missing.
  • The report does not show the required recoding outputs.

If your submission is marked 🟡 Not Ready, revise to address the missing elements and resubmit for reevaluation before the assignment locks.

❌ Missing

No submission.
