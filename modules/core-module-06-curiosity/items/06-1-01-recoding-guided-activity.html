Purpose

Recoding is how analysts turn raw values into clearer categories, labels, and transformed variables that are easier to interpret and use.

In this guided activity, you will learn how vectorized conditional functions work, compare ifelse() and case_when(), and use mutate() to create or overwrite columns in a data frame.

That work is worth doing now because data analysis usually requires more than selecting rows or columns. It often depends on building better variables from the data you already have. These skills will support later work in summarizing, visualizing, and modeling data in R.

Data

No single external dataset required.

This chapter uses dslabs package datasets and built-in examples such as heights, stars, and mtcars, along with small example vectors and data frames created directly in R.

The purpose is to practice recoding, column creation, conditional rules, and checks—not to import a new dataset.

Guiding Questions

  • How does the chapter distinguish between logical subsetting, subassignment, and vectorized conditionals?
  • When is ifelse() enough, and when does case_when() become the clearer choice?
  • Why does the order of rules matter in case_when(), and what can go wrong if the rules are out of order?
  • How does mutate() help you either create a new variable or overwrite an existing column with recoded values?
  • As you follow the code, what checks, memo notes, and inline R code help you confirm that your recoding worked the way you intended?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.
