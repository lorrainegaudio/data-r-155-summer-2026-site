Purpose

Grouped summaries help analysts move from raw rows to useful answers about groups, patterns, and comparisons.

In this guided activity, you will learn how to use count(), summarize(), and group_by() to describe data by category, compute multiple statistics, and interpret what grouped results mean.

That work is worth doing now because later analysis depends on comparing subgroups, not just reporting one number for a whole dataset. These skills also strengthen your Quarto workflow by asking you to organize code, render often, check output, and explain results clearly.

  • Pay close attention to the difference between counting and summarizing, computing more than one statistic at once, grouping by more than one variable, and removing grouping after a summary when needed.

Data

No external dataset required.

This chapter uses built-in example data to practice grouped counts and summaries. The main Guided Activity examples shown in the chapter use mtcars, and you also load dplyr and dslabs as part of the chapter setup.

The purpose is to practice grouping, summarizing, checking grouped output, and explaining what the summaries mean.

Guiding Questions

  • How does the chapter show the difference between using count() for frequencies and using summarize() for custom statistics?
  • When you add group_by(), what changes about the meaning of a result like a mean or a row count?
  • In the examples, which variable is being used to create groups, and which variable is being summarized within those groups?
  • What do you notice when the chapter moves from one grouping variable to combinations of variables?
  • After summarizing, when might you need .groups = "drop" or ungroup(), and how can you check whether grouping is still there?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.
