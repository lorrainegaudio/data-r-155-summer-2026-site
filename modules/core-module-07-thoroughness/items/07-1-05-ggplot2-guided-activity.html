Purpose

In this chapter, you will use ggplot2 to turn data into graphs that are easier to read, compare, and explain.

You will practice the core plotting workflow in R: start with a data frame, map variables inside aes(), and add geom_*() layers to show relationships, distributions, and group comparisons.

This work matters because analysts need to communicate patterns clearly in visual form. You will also keep building reproducible Quarto notes while you create and interpret plots. This guided activity walks you through the basic structure of a ggplot2 graph. You will start with a blank canvas, map variables to visual properties, and then build scatterplots, histograms, boxplots, and labeled graphs in a Quarto document.

Data

Built-in and package-provided datasets are used in this chapter.

This chapter uses datasets that are already available in R or through installed packages, including divorce_margarine from dslabs.

Later practice tasks also use familiar built-in data, such as iris and CO2. The purpose is to practice choosing graph types, mapping variables, and explaining visual patterns.

Guiding Questions

  • How does a ggplot2 graph get built from data, aes(), and geom_*()?
  • What is the difference between starting with a blank ggplot() canvas and adding mapped aesthetics inside aes()?
  • When should you use a scatterplot, a histogram, a boxplot, or a density plot?
  • How do labels and themes make a graph easier to understand?
  • How does Quarto help you keep your code, figures, checks, and explanations together in one reproducible workflow?

You do not submit answers to these questions. Use them to focus your reading, coding, and notes as you work through the chapter.
