Purpose

This assignment collects your Chapter 15 Practice Space work as a Quarto report and workspace file so your simulation code, checks, and short memos are documented in one place.

For this chapter, the uploaded files matter because they show how you used sample(), rnorm(), and replicate() to answer probability questions, interpret the results, and save your work in a reproducible form that can be rerun and reviewed.

File Upload

  • Required files to submit: chapter15_practice.qmd, chapter15_practice.RData
  • File types: .qmd, .RData
  • These files should show:
    • your completed Practice Space work on probability simulation in base R
    • the completed tasks that use set.seed(), sample(), rnorm(), and replicate()
    • the actual code used to simulate die rolls and newborn lengths
    • your checks and short written interpretations
    • your evaluation of fairness and risk
    • your memo responses for each task
  • Based on: Chapter-15_🎲Probabilityfunctions.html

How You Will Be Marked

✅ Complete if all are true

  1. Code + tasks
    • The uploaded chapter15_practice.qmd contains Task 0 through Task 4 for this chapter.
    • The work is clearly for Chapter 15 probability simulation, not a different chapter or assignment.
    • The file is readable, complete, and not blank, cut off, or replaced with unrelated content.
    • The report shows the actual R code used for the chapter tasks.
    • The code includes the required simulation work with sample(), rnorm(), and replicate().
    • The submission includes the chapter-specific task products, including simulation of die rolls, simulation of newborn lengths, repeated weekly simulation, and the reflection on replicate().
  2. Memos
    • The file includes short memos for each required task or step where a memo is requested.
    • The memos answer the chapter’s specific interpretation questions, including comments about:
      • whether the die looks roughly fair
      • what the simulated distributions should look like
      • whether longer blankets seem needed
      • what replicate() was useful for
    • The memos explain what the simulation shows, not just what code was run.
    • If outside help was used, that help is briefly disclosed in a memo.
  3. Reproducibility / chapter-specific requirements
    • The submission includes chapter15_practice.RData with the chapter objects saved from the completed practice work.
    • The work uses the required reproducibility step with set.seed() before simulation code.
    • The code includes appropriate checks for the simulated results, such as table(), mean(), any(), sum(), head(), or other fitting checks.
    • The requested objects are created with the exact names used in the tasks.
    • The files document the chapter workflow clearly enough that the simulation work can be reviewed and rerun.

🟡 Not Ready if any are true

  • No attachment.
  • Wrong artifact.
  • The file cannot be opened.
  • Only one required file is submitted.
  • The uploaded work is blank, incomplete, cut off, or unrelated to Chapter 15.
  • The required simulation code, checks, or memo responses are missing.
  • The .RData file does not contain the needed Chapter 15 objects.

If your submission is marked 🟡 Not Ready, revise to address the missing elements and resubmit for reevaluation before the assignment locks.

❌ Missing

No submission.

Before You Submit

  • Confirm your Quarto file is named chapter15_practice.qmd.
  • Confirm your workspace file is named chapter15_practice.RData.
  • Confirm both files are uploaded.
  • Confirm the Quarto file includes Task 0 through Task 4.
  • Confirm the file includes code using sample(), rnorm(), and replicate().
  • Confirm the file includes checks such as table(), mean(), any(), and sum().
  • Confirm your memos explain what the simulation results suggest in context.
