Purpose

Probability is the language we use to talk about uncertainty, variation, and risk in real data.

In this guided activity, you will use R to simulate outcomes, inspect patterns, and judge whether results look plausible, fair, or unusual.

That work is worth doing now because it helps you move beyond memorizing formulas and start using computation to think like an analyst. The skills in this chapter will support later work in statistics, data analysis, and decision-making whenever outcomes are not perfectly predictable.

Guided Activity

  • Time estimate: 1 hr 30 min
  • Do: Monday
  • Open: Chapter-15_🎲Probabilityfunctions.html
  • Use: RStudio, a Quarto notes file, base R probability and simulation functions, and simulated outcomes created directly in R.
  • Have ready: Your course folder open in RStudio, a new Quarto document started, and enough time to run simulations more than once so you can compare results.
  • Produce: A Chapter 15 Quarto notes file with follow-along code, simulation output, checks, memo notes, and short interpretations of probability results.

Chapter

  • Chapter-15_🎲Probabilityfunctions.html
  • Focus on the main guided activity sections that introduce simulation with sample(), rnorm(), and replicate().
  • Pay close attention to how the chapter checks and interprets simulated results. The goal is not just to generate random values, but to explain whether the results make sense.

Data

No external dataset required.

This chapter uses simulated data created in R during the chapter. You will generate outcomes such as coin flips, dice rolls, and simulated baby lengths directly in R.

The purpose is to practice probability functions, simulation logic, repeated trials, and interpretation of uncertain outcomes.

Guiding Questions

  • How does the chapter show the difference between using sample() for discrete outcomes and rnorm() for continuous outcomes?
  • What choices in the code change the simulation you get, such as the values being sampled, replacement, or probability weights?
  • What checks help you decide whether a simulated result looks reasonable, fair, or unusual?
  • What does repeating a simulation help you notice that a single run might hide?
  • As you follow the code, how does the chapter move from generating random values to making an evidence-based interpretation?

You do not submit answers to these questions, but use them to focus your reading, coding, and memo notes as you work through the Guided Activity.

Before You Leave This Page

  • Confirm that you can use rnorm() to simulate continuous values.
  • Confirm that you can use replicate() to repeat a simulation.

  • Render your Quarto file early enough to catch errors before Practice Space.
